# クラウドネイティブ教科書の位置づけ  
（学習パスにおける役割）

本ドキュメントでは、  
**クラウドネイティブ教科書が学習パス全体の中でどこに位置づけられる教材なのか**  
を整理します。

本教科書は、特定のツール操作や設定手順を教えることを目的としたものではありません。  
クラウドネイティブという技術領域を学ぶ際に、  
**「なぜその設計になるのか」「どこに責任を置いているのか」**  
といった前提や文脈を理解するための土台を提供することを目的としています。

---

## 学習パスにおける位置づけ（概念図）

```text
┌─────────────────────────────────┐
│          実務・現場運用          │
│  （判断・設計・トラブル対応）     │
│                                 │
│  ・設計判断                      │
│  ・障害対応 / トラブルシュート    │
│  ・構成変更 / スケール            │
└───────────────▲─────────────────┘
                │
                │「知識をどう使うか」
┌───────────────┴─────────────────┐
│    Linux Foundation 教材 / 認定  │
│        （KCNA など）             │
│                                 │
│  ・公式な知識体系                │
│  ・標準コンポーネント理解         │
│  ・試験範囲としての整理           │
└───────────────▲─────────────────┘
                │
                │「なぜそうなっているか」
┌───────────────┴─────────────────┐
│      クラウドネイティブ教科書     │
│        （本教材の位置）           │
│                                  │
│  ・前提・考え方                   │
│  ・設計思想 / 文脈                │
│  ・責務の分離                     │
│  ・なぜその設計になるのか         │
└───────────────▲─────────────────┘
                │
                │「土台となる理解」
┌───────────────┴─────────────────┐
│        Linux / インフラ基礎      │
│                                 │
│  ・OS の基本                     │
│  ・ネットワークの基礎             │
│  ・サーバの基本的な挙動           │
└─────────────────────────────────┘
```

* 各レイヤの役割
**Linux / インフラ基礎
-OS、ネットワーク、サーバの基本的な挙動を理解する層
- すべての土台となる知識

**クラウドネイティブ教科書（本教材）
- クラウドネイティブが前提としている世界観を整理する
- 「操作」ではなく「考え方」「設計の前提」を扱う
- Kubernetes やマネージドサービスの設計意図を理解するための層

** inux Foundation 教材 / 認定（KCNA 等）
- 公式に整理された知識体系を学ぶ
- 標準コンポーネントや用語を体系的に理解する
- 認定試験という形で知識を確認する層

**実務・現場運用
- 実際のシステム設計・運用・障害対応を行う層
- 正解のない判断を求められるフェーズ

**本教科書の立ち位置について
本教科書は、
- Linux Foundation の教材や認定を置き換えるものではありません
- 認定試験対策を目的とした教材でもありません

むしろ、
公式教材や認定を「腹落ちした状態」で学ぶための前段
として位置づけています。

いきなりツール操作や用語暗記に進むのではなく、
なぜその設計が採用されているのか
どこまでを人が考え、どこからを仕組みに委ねているのか
を理解した上で次の学習に進むことで、
知識が単なる暗記ではなく、判断に使えるものになることを目指しています。

**想定している学習の流れ（例）
- Linux / インフラの基礎を学ぶ
- クラウドネイティブ教科書で前提・考え方を整理する
- Linux Foundation の公式教材・認定で知識を体系化する
 -実務やハンズオンを通じて判断力を磨く

本教科書は、この流れの 「クラウドネイティブ教科書で前提・考え方を整理する」 を担う教材です。
